This quickstart takes you from a new OpenMIA account to a visible trace in **Observation**.

## Prerequisites

- Access to [app.openmia.ai](https://app.openmia.ai).
- An OpenMIA Auth account.
- Permission to create an Observation Project.
- A small AI runtime payload, or an application that can call an OpenAI-compatible endpoint.

## Create your first Project

<Steps>
  <Step title="Open Observation">
    Sign in to [app.openmia.ai](https://app.openmia.ai), then open **Observation**.
  </Step>
  <Step title="Create a Project">
    Go to **Projects**, click **Create**, enter a Project name, and finish the drawer.
  </Step>
  <Step title="Select the Project">
    Use the Project selector in the page header. Select the Project you just created.
  </Step>
</Steps>

<Info>
  Observation pages use the current Project Scope. If no Project is selected, most modules show an empty state so you can choose a Project first.
</Info>

## Connect data

The fastest path is to create a **Custom JSON** integration and upload or POST one JSON trace.

<Steps>
  <Step title="Create an integration">
    Open **Integrations**, click **Create**, choose **JSON / Webhook**, then choose **Custom JSON**.
  </Step>
  <Step title="Copy the ingest key">
    The plaintext ingest key appears only once. Store it in a secure user-level or server-level secret store.
  </Step>
  <Step title="Send a trace">
    Use the webhook endpoint with your integration key.

    ```bash
    curl -X POST "https://app.openmia.ai/api/observation/webhooks/custom_json?payload_type=json" \
      -H "Authorization: Bearer omia_ing_xxx" \
      -H "Content-Type: application/json" \
      --data '{
        "trace_id": "quickstart_trace_001",
        "name": "Quickstart agent run",
        "status": "success",
        "input": { "question": "What can OpenMIA show me?" },
        "output": { "answer": "Trace, span, raw payload, and audit evidence." },
        "spans": [
          {
            "span_id": "span_001",
            "name": "Agent run",
            "type": "agent",
            "status": "success"
          }
        ]
      }'
    ```
  </Step>
</Steps>

## Inspect the result

Open **Traces** and click your trace. You should see:

- trace metadata and status
- the span tree
- input and output
- related raw payload and import job metadata
- feedback and audit actions, depending on your role
